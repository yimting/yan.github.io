    2. 返回值类型 if else 可不一样 混合类型表达式

    1. 大括号中的多条 语句
    2. 返回值是 : 最后一条语句
    3. 块表达式的值是最后一个表达式的值
    4. 赋值语句的值是unit类型的，

集合

====> 集合是一种可以用来存储各种存储对象的数据的容器 < ========

可变 与 不可变集合

不可变 的 会生成一个新的集合

可变集合 在 原集合中进行修改

可变集合可以在适当的地方 被更新或者扩展, 可以对元素进行 增删改等操作

不可变集合, 永远不会改变 , 不过仍然可以模拟 增删改,

​ 但是这些操作将在每种情况下 ==返回一个新的集合 , 同时保证原来的集合不发生改变==

Scala List ( 列表 ) 其值是不可变的, 其中可存放重复对象, 其次List具有链接表结构
Scala Set ( 集合 ) 无序的, 且不重复
Scala Map ( 映射 ) 映射 ( K , V ) 对的集合 , K 不允许重复
Scala Tuple (元组 ) 元组是==不同类型== 值的==集合==
Scala Iterator (迭代器 ) 不是容器, 可以逐一访问容器中的值的方法
Scala Option ( Option[T] ) 表示有可能包含值的容器, ==( )==表示,或者不包含==None==

隐式(implicit)

隐式参数 隐式转换 隐式调用函数

扩展已有类的功能, 可以是系统定义的类, 还是自己的类, 都可以通过隐式转换 扩展现有的功能

如何定义 :

标记规则:

隐式参数

​ 当我们在定义方法时, 可以将最后一个参数列表标记为implicit , 表示该组参数是隐式参数 , 一个方法只会有一个隐式参数列表, 置于方法的最后一个参数列表

​ 如果方法有很多隐式参数, 只需要一个 implicit 修饰即可, 当调用包含隐式参数的方法时 , 如果上下文存在合适的 隐式值 , 那么系统会自动更改组参数填充合适的值

​ 如果不存在的话会抛出异常, 当然标记为隐式参数的我们可以手动

隐式地转换类型

使用隐含转换将变量转换成预期的类型是编译器最先使用 implicit 的地方。这个规则非常简单，当编译器看到类型X而却需要类型Y，它就在当前作用域查找是否定义了从类型X到类型Y的隐式定义
